一、Demo 简介

阿嬷的频道是一个专为老年人设计的智能点播网站。奶奶外婆想看莆仙戏、歌仔戏、南音、木偶戏，或者想听音乐、看电视剧和电影，可以输入中文、方言文字或习惯叫法，也可以用普通话或方言语音说出戏名，系统通过 4 层语音识别降级架构（百度语音→科大讯飞→Google Web Speech→FunASR）完成语音转文字，再通过 606 条深度泛化词条库进行本地词典翻译、别名映射与模糊匹配，在节目库中找到匹配的内容，显示封面图让老人家确认后播放。

当前版本已部署为 GitHub Pages 在线版本，同时提供离线 ZIP 包。语音识别通过阿里云函数计算（FC）代理实现，前端不暴露任何 API 密钥。404 万条深度泛化词条覆盖全称/简称、拼音谐音（普通话+莆田口音+闽南口音）、方言口语、角色名、剧情关键词、歌词台词、演员名等 7 大类型，确保老人各种说法都能匹配到正确节目。

产品形态是一个单文件网页应用（HTML），无需安装、无需注册、无广告，打开即用。采用移动优先的响应式设计，完美适配手机、平板、电脑和电视大屏。字号自适应、高对比度、大按钮，专为老年人视力和操作习惯量身打造。图片采用 480px 等比例缩放，配合懒加载，在手机和平板上也能快速浏览。

当前 Demo 包含 7 个分类、35 个精选节目：

莆仙戏（5部）：春草闯堂、状元与乞丐、江梅妃、新亭泪、秋风辞
歌仔戏（5部）：陈三五娘、梁山伯与祝英台、吕蒙正、薛平贵与王宝钏、王金龙与苏三
南音（5首）：八骏马、梅花操、百鸟归巢、三千两金、直入花园
木偶戏（5部）：小沙弥下山、驯猴、元宵乐、大名府、雷万春打虎
电视剧（5部）：射雕英雄传、红楼梦、三国演义、水浒传、西游记
音乐（5首）：东方红、没有共产党就没有新中国、打靶归来、最炫民族风、小苹果
电影（5部）：地道战、铁道游击队、英雄儿女、人到中年、庐山恋

每个节目都有 AI 生成的海报卡片，35 个节目均配置 6 秒程序合成功能演示音频，部分节目按类别复用演示音轨。长辈可以通过文字输入、语音输入、听节目名选台或直接点击节目海报来点播。演示音频仅用于验证点播与播放器交互，不是节目原声。

当前版本为比赛演示版。音视频内容采用占位演示，不提供真实版权内容播放；正式上线需要接入公共版权、授权内容或地方文化馆资源。

二、Demo 创作思路

灵感来源：

我的灵感来自我的阿嬷。我看到她用手机找戏曲时特别费劲：她识字不多，不会打字，普通话也说不准，平常交流只用莆仙本地方言。每次想看一段戏，她都需要别人帮忙翻找。

后来我发现，这不仅是我阿嬷的烦恼，闽南地区想看歌仔戏的爷爷奶奶们也有同样的困扰——现在的视频平台基本只能用文字搜索，不支持方言。屏幕上字小、广告又多，老人家经常点错，找不到想看的原版内容，最后只能无奈放弃。

放眼望去，单单福建就有不下十种方言，全国还有粤语、潮汕话、客家话、吴语等数不胜数的乡音。很多生活在农村的老人读书少，打字困难，日常更是只会说方言。如果能做一个能识别方言语音、文字、家庭习惯叫法和近似音的智能系统，让他们用自己的家乡话就能轻松点播想看的节目，一定能大大改善老人们的生活便利性，提升他们的幸福感。

想解决的问题：

不会打字：老年人不熟悉拼音和手写输入
方言障碍：很多长辈普通话说不准，更习惯只说家乡方言（莆仙话、闽南话等）
操作复杂：打开App、搜索、避开广告、辨认结果、点播放，步骤太多
找不到原版：搜索结果混杂各种翻拍、剪辑版本，找不到正宗原声
传统文化被数字鸿沟挡在门外：莆仙戏（国家级非遗）、歌仔戏（福建省级非遗）、南音（世界级非遗）等面临观众老龄化和年轻观众断层

为什么做这个方向：

选择用"方言词典+4 层语音识别降级+智能模糊匹配"的方案。AI 在这里不是冷冰冰的"合成语音发声筒"，而是"能听懂方言语音、识别方言文字、家庭习惯叫法和近似音文本的智能点播台"。通过阿里云 FC 代理百度语音（10 万次/天免费）和科大讯飞（500 次/天免费），结合 FunASR 开源模型和 Google Web Speech 浏览器接口，在国内任何网络环境下都能实现稳定的语音识别。

四、当前产品体验

进入网页后，顶部是一个大字号搜索区，包含语言选择（莆仙话/闽南话/中文）、输入框、麦克风、点播按钮和听节目名选台按钮。搜索栏采用类似搜索引擎的简洁设计，顶部栏会在页面向上滑动时自动隐藏，往下拉时自动显示，让老人浏览节目时有更大空间。搜索栏留白充足，方便查看语音识别结果或打字输入内容。

节目分类标签包含：全部、莆仙戏、歌仔戏、南音、木偶戏、电视剧、音乐、电影。默认显示"全部"分类，下面是节目海报列表，每张海报等比例缩放为 480px 宽度（16:9 比例），在手机和平板上快速加载。每个海报右下角有圆形播放按钮，长辈可以直接点图片或按钮选节目。

当输入或语音识别到内容后，系统会：

语音识别：4 层降级架构——第 1 层阿里云 FC 代理百度语音（国内快速稳定）→ 第 2 层科大讯飞（支持方言）→ 第 3 层 Google Web Speech（浏览器内置）→ 第 4 层 FunASR（阿里云 FC 容器部署）→ 全部失败则提示用户打字输入。
清洗口语表达，比如"我要看""帮我找""播放一下""给我放"。
修正常见误识别，比如"秋分"→《秋风辞》、"春草闯唐"→《春草闯堂》、"状元乞丐"→《状元与乞丐》。
通过 606 条深度泛化词条库进行匹配，覆盖：全称/简称、拼音谐音（普通话+莆田口音+闽南口音）、方言口语、角色名、剧情关键词、歌词台词、演员名。
按优先级依次尝试：清洗后的翻译文本 → 翻译文本 → 清洗后的原始文本 → 原始文本 → 拼音近似。
匹配到节目后，弹出居中确认卡，显示节目图片、名称、简介和"就是这个""换一个"按钮。
点击"就是这个"后进入播放器。
如果图片加载失败，系统会自动显示分类占位图，避免出现破图。完成一次点播后，可以继续点播其他节目，不需要刷新网页。

五、四种点播方式

当前作品实现了四种互补的点播路径，覆盖不同能力和习惯的长辈：

方式一：4 层语音识别输入（推荐）

点击麦克风按钮后，系统自动录音 5 秒，然后依次尝试 4 层语音识别服务：
第 1 层：阿里云 FC 代理百度语音（每天 10 万次免费额度，国内访问快速稳定）
第 2 层：科大讯飞语音听写（支持闽南话识别，每天 500 次免费额度）
第 3 层：Google Web Speech（浏览器内置，无需额外配置）
第 4 层：FunASR（阿里云 FC 容器部署，支持实时语音识别）
全部不可用时，友好提示用户直接打字输入。

整个过程中，搜索栏会实时显示当前使用的识别层和状态，老人家能清楚看到系统在做什么。所有 API 密钥存储在阿里云 FC 环境变量中，前端不暴露任何密钥信息。

方式二：搜索栏文字输入

在网页顶部的搜索栏直接输入中文节目名称（如"春草闯堂"）、方言文字（如莆仙话/闽南话），或输入不完整的戏名（如"状元乞丐""秋分"等），系统通过 606 条深度泛化词条库自动修正常见误识别和口音差异，通过多级候选搜索匹配节目，弹出确认卡后播放。文字搜索和词典匹配在本地完成，无需联网。

方式三：听节目名选台

点击"听节目名选台"按钮后，系统用中文语音朗读每个节目名称。长辈听到想看的节目后点击"就是这个"确认，不想听的点击"下一个"跳过，也可以点"上一个"返回。包含全部 35 个节目，对于看字吃力或不习惯打字的长辈，这是最简单的操作方式。

方式四：浏览海报图片直接播放

页面下方按分类展示所有节目海报，每张海报反映该节目的剧情内容和风格，图片等比例缩放适合手机和平板快速浏览。长辈浏览到感兴趣的节目后，点击海报或右下角的播放按钮，系统直接进入播放器播放。适合喜欢浏览和自主选择的用户。

适老化界面与连续点播：整体字体加大，按钮触控区域加大，高对比度显示，简洁搜索栏，滚动时顶部自动隐藏/显示，适配手机、平板和电脑。播放一个节目后可以返回继续点播其他节目，状态正确清理，无需刷新页面。

六、606 条深度泛化词条库

词条库是本作品方言匹配能力的核心，从最初的 150 条迭代到 606 条，覆盖 35 个节目的 7 大类型词条：

全称/简称（节目名、角色名）：说一个词就能搜，如"春草"→春草闯堂
拼音谐音（普通话+莆田口音+闽南口音）：老人发音不准也能匹配，如"新品类"→新亭泪
方言口语（莆田/闽南老人常说的叫法）：方言直接匹配，如"sih ping kui"→薛平贵
角色名（主角、配角、反派、相关人物）：说角色也能搜到节目，如"郭靖"→射雕英雄传
剧情关键词（关键情节、场景、主题）：说情节也能搜到，如"化蝶"→梁山伯与祝英台
歌词/台词（经典台词、歌词首句、副歌）：说歌词也能搜到，如"日落西山红霞飞"→打靶归来
演员（扮演者）：说演员搜节目，如"张瑜"→庐山恋

各分类词条分布：莆仙戏 132 条、歌仔戏 116 条、电视剧 103 条、电影 83 条、音乐 65 条、木偶戏 58 条、南音 49 条。

七、4 层语音识别降级架构

针对国内网络环境和方言识别需求，设计了 4 层自动降级的语音识别架构：

第 1 层：阿里云 FC → 百度语音识别
通过阿里云函数计算（FC）代理调用百度短语音识别 REST API。百度语音每天提供 10 万次免费额度，国内访问快速稳定。FC 函数自动管理百度 access_token 缓存，前端只需发送录音数据，无需处理鉴权。密钥存储在 FC 环境变量中，前端零暴露。

第 2 层：阿里云 FC → 科大讯飞语音听写
百度识别失败时自动降级到科大讯飞 WebSocket API。讯飞支持闽南话识别（language: minnan），对新用户每天提供 500 次免费额度。通过 HMAC-SHA256 签名鉴权，音频分帧发送，支持动态端点检测（VAD）和中间结果返回。

第 3 层：Google Web Speech
浏览器内置的语音识别接口（SpeechRecognition / webkitSpeechRecognition），无需任何服务器配置，在 Chrome/Edge 中可用。适合海外网络或有科学上网环境的用户。

第 4 层：FunASR（阿里云 FC 容器部署）
基于阿里达摩院开源的 FunASR 实时语音识别模型，部署在阿里云 FC 容器镜像上。支持 2pass 模式（在线+离线），前端将 WebM 音频转为 16kHz PCM 后通过 WebSocket 发送。在所有云服务不可用时提供最后一道保障。

全部失败：友好提示用户"语音识别暂时不可用，请打字输入节目名"。

前端实现：点击麦克风后统一进入 startVoiceWithFallback() 函数，自动录音 5 秒，按层尝试，每层有独立超时控制（百度/讯飞 8 秒、Google 6 秒、FunASR 10 秒），搜索栏实时显示当前识别状态。

八、创新点

第一，把"方言"变成入口，而不是障碍。

很多适老化产品只解决字体和按钮问题，但《阿嬷的频道》进一步关注老人真实表达方式。606 条深度泛化词条库覆盖 7 大类型，4 层语音降级架构确保在国内任何网络环境下都能用语音点戏。长辈说不标准普通话、只会方言，都不应该成为他们使用数字内容的门槛。

第二，4 层语音识别降级，零密钥暴露。

通过阿里云 FC 代理，所有语音识别 API 的密钥（百度、讯飞）存储在服务端环境变量中，前端 HTML 不包含任何密钥信息。4 层自动降级确保服务可用性：一层不可用自动切换下一层，用户全程无感知。

第三，以非遗为起点，连接日常娱乐。

作品不是把非遗做成冷冰冰的资料库，而是把它放回长辈日常生活中。阿嬷可以看莆仙戏、歌仔戏、木偶戏，也可以听南音、看经典电视剧、电影和音乐节目。

第四，聚焦福建，又具备综合扩展能力。

Demo 以福建地方戏曲和传统音乐为核心，同时加入经典影视和音乐内容。后续可以扩展全国戏曲、多地方言和更多娱乐节目，成为长辈专属的娱乐点播入口。

第五，从"搜索"升级为"点播"。

普通网页让用户自己搜索，而这个作品更像一个懂阿嬷说话方式的频道助手。它把"说一句话→4 层语音识别→606 条词条匹配→看图片确认→进入播放器"做成完整路径，降低了使用门槛，也增强了家庭陪伴感。

九、版权与素材说明

为避免未经授权传播影视、戏曲和音乐内容，本 Demo 不内置原节目音视频。35 个节目均配置 6 秒程序合成功能演示音频（共 12 种不同内容，部分节目按类别复用演示音轨），演示音频仅用于验证点播与播放器交互，不是节目原声。播放器页面和确认卡页面均标注"语音为AI合成，仅供功能演示，正式版本将在取得授权后接入合法音视频源"。

当前版本面向 35 个精选节目及已录入的家庭习惯叫法进行封闭域匹配，不作为任意节目或开放内容搜索工具。

素材来源与许可信息：

节目海报：由 AI 图像生成工具制作，为原创素材
演示音频：35 个节目均配置 6 秒程序合成功能演示音频（共 12 种不同内容，部分节目按类别复用），为原创素材
不包含从视频网站、电视或网盘截取的任何第三方音视频片段
不使用知名旋律的翻唱、变速或重新演奏
不使用明星、演员或知名艺术家的仿声音频
不使用现代剧团演出录音

效率提升：把"打开App→打字→搜索→避开广告→辨认结果→播放"简化为"说一句话或输入文字→确认→播放"。

生活品质：长辈能自己找节目，更有参与感和掌控感。家人少一些代搜，多一些陪伴。

代际沟通：科技成为了解长辈喜好的桥梁。孩子通过这个作品了解奶奶喜欢什么。

文化传承：让莆仙戏（国家级非物质文化遗产）、歌仔戏（福建省非物质文化遗产）、南音（人类非物质文化遗产）等传统文化通过新科技触达老年群体，方言不被数字鸿沟阻挡。同时让年轻一代通过这个作品重新认识和欣赏传统文化。

消除数字鸿沟：中国有超过 2.6 亿 60 岁以上老年人，其中大量不会使用智能手机。这个项目用最简单的"方言语音+文字+按钮"方式，让不会打字、只会方言的老年人也能享受数字时代的文化娱乐权益，不让任何一位阿嬷被遗忘在数字时代之外。

十二、技术架构

前端：单文件 HTML 应用，内置 606 条莆仙话词条 + 501 条闽南话词条，图片懒加载，响应式设计
语音识别：4 层降级架构（百度→讯飞→Google→FunASR）
后端代理：阿里云函数计算（FC）Python 3.9 函数，代理百度语音和科大讯飞 API
FunASR：阿里云 FC 容器镜像部署，支持 2pass 实时语音识别
部署：GitHub Pages（前端）+ 阿里云 FC（语音代理 + FunASR）
安全：所有 API 密钥存储在 FC 环境变量，前端零密钥暴露
图片优化：480px 等比例缩放（16:9），懒加载，适配手机和平板快速浏览

十三、最后

《阿嬷的频道》想做的事情很简单：

让长辈不用学复杂技术，也能打开自己的频道。

让他们熟悉的乡音、戏曲和节目，被更容易地找到。

也让年轻人在帮阿嬷点播的过程中，重新看见地方文化的温度。
